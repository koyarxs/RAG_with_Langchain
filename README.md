# Retrieval Augmented Generation (RAG) with LangChain

Proyecto práctico utilizando IBM Granite Models y LangChain para construir un sistema RAG (Retrieval Augmented Generation).

---

## 📌 Descripción

Este laboratorio demuestra cómo crear un sistema capaz de:

- Recuperar información desde una base de conocimiento
- Generar respuestas contextualizadas utilizando IA
- Utilizar embeddings y búsqueda semántica
- Integrar modelos IBM Granite con LangChain

---

## 🧠 Tecnologías utilizadas

- Python
- Google Colab
- LangChain
- IBM Granite Models
- Replicate
- ChromaDB
- HuggingFace Embeddings
- Vector Databases
- RAG Architecture

---

## 🚀 ¿Qué es RAG?

RAG (Retrieval Augmented Generation) es una arquitectura que combina:

1. Recuperación de información relevante
2. Generación de respuestas mediante IA

Esto permite que los modelos respondan utilizando información específica obtenida desde documentos o bases de conocimiento.

---

## 📂 Contenido del proyecto

- Configuración del entorno
- Instalación de dependencias
- Creación de embeddings
- Almacenamiento vectorial
- Recuperación semántica
- Generación de respuestas con IA
